## 1. 사업 개요

### 1.1 한 줄 요약

> **"공공 입찰 시장의 정보 비대칭을 AI로 해소하는 B2B SaaS 플랫폼"**

### 1.2 사업 배경

대한민국 공공 조달 시장은 연간 **약 180조 원** 규모(조달청 2025년 기준)로, 국가 및 지방자치단체·공공기관의 모든 물품·공사·용역 계약이 나라장터(G2B)를 통해 공개 집행됩니다. 그러나 다음과 같은 구조적 문제로 인해 중소·중견 기업들이 실질적인 입찰 기회를 포착하지 못하고 있습니다.

- **정보 과부하**: 나라장터에는 하루 평균 3,000건 이상의 공고가 게시됨
- **수작업 모니터링**: 담당자가 매일 직접 검색·확인하는 비효율적 프로세스
- **기회 손실**: 마감일 임박 공고를 뒤늦게 파악하여 입찰 기회 상실
- **시장 인텔리전스 부재**: 경쟁 현황·낙찰 통계 분석 도구 부재

BidSight는 이러한 문제를 해소하는 **AI 기반 공공입찰 인텔리전스 플랫폼**입니다.

### 1.3 핵심 가치 제안

| 가치 | 설명 |
|------|------|
| **시간 절약** | 하루 3시간 수작업 → 분 단위 자동 알림 |
| **기회 극대화** | 조건 맞는 공고를 놓치지 않고 100% 포착 |
| **시장 인사이트** | 기관·업종별 통계, 낙찰 패턴 분석 |
| **의사결정 지원** | AI 낙찰률 예측으로 입찰 전략 수립 지원 |

## 4. 제품 및 서비스

### 4.1 핵심 기능 (MVP, 현재 구현 완료)

#### 공고 검색 및 필터
- 나라장터 공고 자동 수집 (1일 1회, Vercel Cron)
- 키워드·상태·지역·업종·예산 범위 필터링
- 실시간 검색 (300ms 디바운스)
- 페이지네이션 (20건/페이지)

#### 맞춤 알림
- 알림 규칙 생성 (키워드 / 필터 유형)
- 조건 매칭 시 이메일 알림 발송 (Resend)
- 카카오 알림톡 발송 (확장 준비 완료)
- 알림 발송 이력 조회

#### 즐겨찾기 관리
- 관심 공고 즐겨찾기 등록/해제
- 즐겨찾기 목록 일괄 관리

#### 분석 리포트
- 기간별 공고 통계 (총 건수, 총 예산, 기관 수)
- 상태별 분포 (진행중/마감/결과발표)
- 발주 기관 TOP 10
- 업종별 TOP 10

### 4.2 로드맵 기능 (향후 개발)

| Phase | 기능 | 목표 시기 |
|-------|------|----------|
| Phase 1 | 배치 파이프라인 고도화 (QStash, 10분 주기 수집) | 2026 Q2 |
| Phase 2 | 한국어 형태소 검색 (OpenSearch + Nori) | 2026 Q2 |
| Phase 3 | 고급 분석 대시보드 (차트, 파티셔닝) | 2026 Q3 |
| Phase 4 | AI 낙찰률 예측 모델 (XGBoost) | 2026 Q3~Q4 |
| Phase 5 | 멀티테넌시 & 구독 결제 (토스페이먼츠) | 2026 Q4 |

### 4.3 기술 스택

```
Frontend  : Next.js 16 (App Router) · TypeScript · TailwindCSS v4 · shadcn/ui
Backend   : Next.js API Routes · Zod 입력검증 · Supabase Auth + RLS
Database  : Supabase (PostgreSQL) · pg_trgm · Row Level Security
외부 API  : 나라장터 (공공데이터포털) · Resend (이메일) · Kakao (알림톡)
인프라    : Vercel (자동 배포) · Supabase (Seoul 리전) · GitHub
```

## 6. 마케팅 및 영업 전략

### 6.1 목표 고객 (Persona)

**Persona A — "바쁜 영업 담당자"**
- 직함: 중소 IT 기업 영업부 팀장
- 고민: 입찰 공고 모니터링에 하루 2~3시간 낭비
- 니즈: 자동 알림, 빠른 공고 확인

**Persona B — "데이터 기반 전략가"**
- 직함: 건설사 사업개발 이사
- 고민: 어느 기관을 공략해야 하는지 데이터 근거 부족
- 니즈: 기관·업종 통계, 낙찰 이력 분석

**Persona C — "입찰 컨설턴트"**
- 직함: 프리랜서 입찰 컨설턴트
- 고민: 다수 고객사의 알림을 동시에 관리
