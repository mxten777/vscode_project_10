# BidSight 메시지 가이드

이 문서는 BidSight를 설명할 때 문구 톤이 흔들리지 않도록 기준 문안을 모아둔 문서입니다.

목적은 3가지입니다.

- 랜딩과 소개 문구가 같은 메시지를 말하게 만든다.
- 고객, 파트너, 내부 영업 담당자가 짧게 설명할 때 기준 문장을 제공한다.
- 화면 중간중간 안내 문구를 같은 흐름 언어로 맞춘다.

## 1. 랜딩용 소개 문구

### Hero 제목

필요한 공고만 먼저 보고,
판단 근거까지 순서대로 확인하는 입찰 워크플로우

### Hero 보조 문장

BidSight는 공공입찰 공고를 많이 보여주는 서비스가 아니라,
오늘 먼저 검토할 공고를 빠르게 좁히고, 참여 판단 근거를 확인하고,
놓치지 않게 추적하도록 돕는 입찰 검토 지원 서비스입니다.

### Hero 옆 요약 카드에 넣을 메시지

- 먼저 하는 일: 후보 공고 좁히기
- 다음에 하는 일: 상세에서 판단 근거 확인
- 마지막에 하는 일: 저장, 알림, 분석으로 정리

### 핵심 효익 3줄

- 검색 시간을 줄입니다.
- 중요한 공고를 놓치지 않게 합니다.
- 참여 판단 근거를 더 빨리 정리하게 합니다.

### CTA 문구

- 무료로 시작하기
- 지금 검토 흐름 보기
- 필요한 공고만 먼저 보기

## 2. 고객 설명용 짧은 버전

### 한 문장 소개

BidSight는 공공입찰 공고를 자동으로 모으고, 필요한 공고만 빠르게 찾고, 놓치지 않게 알려주며, 참여 판단 근거까지 정리해 주는 입찰 지원 서비스입니다.

### 3문장 버전

BidSight는 나라장터 공고를 자동으로 모으고, 먼저 검토할 공고를 빠르게 추리는 서비스입니다.
사용자는 검색, 상세 검토, 저장과 알림, 분석 화면을 한 흐름으로 이어서 볼 수 있습니다.
그래서 공고를 뒤지는 시간과 누락 위험을 줄이고, 참여 여부 판단을 더 빠르게 할 수 있습니다.

### 고객 미팅 첫 문장

"이 서비스는 공고를 많이 보여주는 제품이 아니라, 어떤 공고를 먼저 보고 어떻게 놓치지 않을지를 정리해 주는 도구입니다."

### 고객이 바로 이해해야 할 질문

- 오늘 먼저 봐야 할 공고가 무엇인가
- 왜 이 공고를 우선 검토해야 하는가
- 어떻게 놓치지 않고 추적할 수 있는가
- 참여 판단 근거를 어디서 확인하는가

## 3. 투자자·파트너 설명용 짧은 버전

### 3문장 버전

공공입찰 실무자는 매일 많은 공고를 직접 확인하지만, 무엇을 먼저 봐야 하는지와 무엇을 놓치지 말아야 하는지를 빠르게 정리하기 어렵습니다.
BidSight는 공고 수집, 선별, 추적, 분석을 한 흐름으로 묶어 반복 검토 비용을 줄이는 입찰 운영 도구입니다.
핵심 가치는 검색 시간 절감, 공고 누락 감소, 검토 우선순위 정리, 참여 판단 지원입니다.

### 한 줄로 더 짧게 말할 때

BidSight는 공공입찰 검토 순서를 정하고, 중요한 공고를 놓치지 않게 만드는 워크플로우형 입찰 지원 서비스입니다.

## 4. 화면별 안내 문구 기준

안내 문구는 아래 3요소로만 씁니다.

1. 현재 단계: 지금 사용자가 무엇을 하는 구간인지
2. 다음 행동: 다음으로 어디로 가야 하는지
3. 기대 결과: 그 행동을 하면 무엇이 달라지는지

### 랜딩

- 현재 단계: 이 서비스가 어떤 흐름을 제공하는지 이해하는 단계입니다.
- 다음 행동: 로그인 후 공고를 좁히는 화면으로 들어갑니다.
- 기대 결과: 기능 목록이 아니라 검토 순서를 이해한 상태로 시작합니다.

### 메인 검색 화면

- 현재 단계: 지금은 후보 공고를 빠르게 좁히는 단계입니다.
- 다음 행동: 상세 화면으로 넘어가 판단 근거를 확인하세요.
- 기대 결과: 먼저 검토할 공고를 빠르게 정할 수 있습니다.

### 공고 상세 화면

- 현재 단계: 지금은 참여 판단 근거를 확인하는 단계입니다.
- 다음 행동: 검토 대상이면 저장하거나 알림으로 연결하세요.
- 기대 결과: 이 공고를 계속 볼지, 추적할지 빠르게 결정할 수 있습니다.

### 즐겨찾기 화면

- 현재 단계: 지금은 다시 볼 공고를 정리하는 단계입니다.
- 다음 행동: 저장한 공고를 다시 열어 판단을 마무리하세요.
- 기대 결과: 검토가 필요한 공고를 잊지 않고 이어서 볼 수 있습니다.

### 알림 화면

- 현재 단계: 지금은 중요한 공고를 놓치지 않는 구조를 만드는 단계입니다.
- 다음 행동: 규칙을 활성화하고 발송 이력을 확인하세요.
- 기대 결과: 반복 검색 없이 필요한 조건의 공고를 계속 추적할 수 있습니다.

### 분석 화면

- 현재 단계: 지금은 전체 흐름 속에서 우선순위를 정리하는 단계입니다.
- 다음 행동: 기관, 업종, 지역 비교로 다음 집중 대상을 정하세요.
- 기대 결과: 개별 공고를 넘어서 어디에 시간을 써야 할지 판단할 수 있습니다.

### 리포트 화면

- 현재 단계: 지금은 검토 결과를 정리하고 공유하는 단계입니다.
- 다음 행동: 기간을 선택하고 PDF 또는 CSV로 내보내세요.
- 기대 결과: 팀이 같은 기준으로 흐름을 공유할 수 있습니다.

### 팀 화면

- 현재 단계: 지금은 검토 기준을 함께 맞추는 단계입니다.
- 다음 행동: 멤버를 초대하고 이후 알림 또는 리포트로 이어가세요.
- 기대 결과: 개인 검토가 아니라 팀 단위 운영으로 연결됩니다.

### 설정 화면

- 현재 단계: 지금은 운영 기본값을 정리하는 단계입니다.
- 다음 행동: 계정, 회사 기준, 플랜 순서로 설정을 점검하세요.
- 기대 결과: 이후 검색, 저장, 알림 흐름이 흔들리지 않게 됩니다.

## 5. 말하면 안 되는 방식

- 기능이 많다는 식의 나열형 설명
- 아직 운영 검증이 덜 된 AI/결제를 전면에 내세우는 설명
- "모든 걸 자동화한다" 같은 과장 표현
- UI가 화려하다는 점만 강조하는 설명

## 6. 항상 유지해야 하는 메시지

- 먼저 볼 공고를 빠르게 추린다.
- 중요한 공고를 놓치지 않게 한다.
- 참여 판단 근거를 더 쉽게 정리하게 한다.
- 팀이 같은 기준으로 검토하게 만든다.
