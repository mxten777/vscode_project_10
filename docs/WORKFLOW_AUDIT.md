# BidSight 핵심 사용자 흐름 점검

이 문서는 현재 화면 구조가 클라이언트의 실제 업무 흐름을 얼마나 잘 지원하는지 점검하고, 다음 UI/UX 개편 순서를 고정하기 위한 문서입니다.

기준은 단순합니다.

- 사용자가 무엇을 해야 하는지 첫 화면에서 바로 보여야 한다.
- 화면은 기능 목록이 아니라 업무 순서대로 이어져야 한다.
- 판단에 필요한 정보가 장식보다 먼저 보여야 한다.

## 1. 기준 사용자 흐름

BidSight의 핵심 사용자 흐름은 아래 순서로 봐야 합니다.

1. 오늘 볼 공고를 찾는다.
2. 후보를 빠르게 선별한다.
3. 상세 화면에서 참여 여부 판단 근거를 본다.
4. 중요한 공고를 저장하거나 알림으로 추적한다.
5. 분석과 리포트로 전체 우선순위를 정리한다.

## 2. 현재 흐름 점검 결과

### A. 메인 목록 화면

현재 역할:

- 공고 검색, 카테고리 탐색, 정렬, 목록 탐색의 출발점

현재 장점:

- 검색과 정렬 기능 자체는 이미 존재한다.
- 목록과 상세로 이어지는 기본 흐름이 있다.
- 요약 지표와 마감 정보가 있어 빠른 훑어보기가 가능하다.

현재 문제:

- Hero, 통계 카드, 카테고리 칩, 트렌딩 키워드, 검색, 목록이 한 화면에서 동시에 강하게 경쟁한다.
- 사용자가 "지금 무엇부터 해야 하는지" 보다 "정보가 많다" 는 인상을 먼저 받는다.
- AI 인사이트와 시각 효과가 핵심 업무 흐름보다 먼저 눈에 들어올 가능성이 있다.
- 검색 결과를 빠르게 선별하는 화면이라기보다, 정보가 풍부한 대시보드처럼 보일 수 있다.

정리:

- 메인 화면은 `대시보드` 보다 `작업 시작 화면` 이어야 한다.
- 첫 시선은 검색, 필터, 우선 검토 대상에 모여야 한다.

### B. 공고 상세 화면

현재 역할:

- 공고 상태, 기본 정보, 낙찰 결과, 보조 분석을 보여줌

현재 장점:

- 상태, 예산, 일정, 낙찰 결과를 확인할 수 있다.
- 즐겨찾기 토글이 상단에 있어 후속 액션으로 이어진다.
- 검토 포인트 요약 카드가 있어 읽는 순서를 어느 정도 안내한다.

현재 문제:

- 상단 시각 밀도와 섹션 수가 많아 판단 포인트가 분산될 수 있다.
- OPEN 공고에서 참고용 투찰가 분석이 비교적 크게 보여 핵심 판단 정보보다 앞에 느껴질 수 있다.
- 기본 정보, 결과, 추천, 유사 사례가 모두 강하게 배치돼 중요도 차이가 약하다.

정리:

- 상세 화면은 `정보 전시` 가 아니라 `참여 검토` 화면이어야 한다.
- 가장 먼저 보여야 할 것은 상태, 마감, 예산, 기관, 결과 데이터 유무다.

### C. 즐겨찾기 화면

현재 역할:

- 관심 공고 목록 관리

현재 장점:

- 단순하고 목적이 분명하다.
- 목록 구조가 가볍고 상세 이동이 쉽다.

현재 문제:

- 저장 이유와 다음 행동이 약하다.
- 단순 보관함처럼 보일 수 있고, 검토 우선순위 관리 도구라는 인상이 약하다.

정리:

- 즐겨찾기는 `저장` 보다 `다시 볼 공고 관리` 성격이 드러나야 한다.

### D. 알림 관리 화면

현재 역할:

- 규칙 생성, 활성화/비활성화, 발송 로그 확인

현재 장점:

- 생성 흐름이 짧고 현재 구조도 비교적 명확하다.
- 빠른 흐름 안내가 있어 사용 맥락을 설명한다.

현재 문제:

- 규칙 생성 폼이 기능 항목 위주라 사용 목적별 템플릿 느낌은 약하다.
- 로그와 규칙은 정리돼 있지만, 어떤 규칙을 왜 만들어야 하는지가 조금 더 드러날 필요가 있다.

정리:

- 알림 화면은 `설정 화면` 보다 `놓치지 않기 위한 추적 도구` 처럼 보여야 한다.

### E. 분석 대시보드

현재 역할:

- 기관, 업종, 지역 기준 통계와 차트 제공

현재 장점:

- KPI 와 차트가 풍부하다.
- 전체, 기관, 업종, 지역으로 분석 축이 나뉘어 있다.

현재 문제:

- 처음 보는 사용자는 어디부터 읽어야 할지 잠깐 멈출 수 있다.
- 차트 수가 많아 `해석` 보다 `조회` 경험에 가까워질 수 있다.
- 실제 업무 질문과 바로 연결되는 요약 문장이 더 필요하다.

정리:

- 분석 화면은 `차트 모음` 이 아니라 `의사결정 요약 화면` 이어야 한다.

## 3. 핵심 병목 정리

현재 가장 큰 병목은 아래 4가지입니다.

1. 메인 화면이 작업 시작 화면보다 정보형 대시보드에 가깝다.
2. 상세 화면에서 판단 우선순위보다 시각적/부가적 정보가 강할 수 있다.
3. 즐겨찾기와 알림이 관리 기능으로 보이고, 업무 후속 흐름으로는 약하다.
4. 분석 화면은 데이터는 많지만 해석 우선순위가 더 필요하다.

## 4. 다음 UI/UX 개편 순서

이 순서대로 진행하는 것이 가장 효율적입니다.

### Step 1. 메인 목록 화면 재설계

목표:

- 첫 진입 시 무엇을 해야 하는지 바로 보이게 한다.
- 검색, 필터, 우선 검토 공고가 가장 먼저 읽히게 만든다.

핵심 변경:

- Hero 비중 축소
- 핵심 KPI 축약
- 필터와 검색 우선 배치
- 목록 카드의 검토 포인트 강화

### Step 2. 공고 상세 화면 재정렬

목표:

- 참여 여부 판단에 필요한 정보가 가장 먼저 오게 만든다.

핵심 변경:

- 상태/마감/예산/기관/결과 유무 우선 배치
- 추천/보조 분석은 후순위로 이동
- 상단 액션을 저장/추적으로 명확화

### Step 3. 즐겨찾기와 알림 흐름 개편

목표:

- 저장과 알림이 각각 어떤 상황에서 필요한지 더 분명하게 만든다.

핵심 변경:

- 즐겨찾기: 다시 볼 공고 관리 중심
- 알림: 놓치지 않기 위한 추적 중심
- 빈 상태와 생성 유도 문구 강화

### Step 4. 분석 화면 요약 중심 재구성

목표:

- 차트를 보기 전에 무엇을 읽어야 하는지 먼저 보이게 한다.

핵심 변경:

- 요약 인사이트 문장 추가
- KPI 와 차트의 읽는 순서 재배치
- 실제 의사결정 질문 기준으로 탭과 레이블 정리

## 5. 바로 다음 작업

1. 메인 목록 화면의 정보 구조를 먼저 다시 짠다.
2. 그 기준에 맞춰 상세 화면을 판단 중심으로 재배치한다.
3. 이후 즐겨찾기/알림, 마지막으로 분석 화면 순서로 진행한다.

## 6. 한 줄 결론

현재 BidSight는 기능은 꽤 갖추고 있지만, 화면 구조가 아직 `실무 시작 → 판단 → 추적 → 분석` 순서로 완전히 정돈되지는 않았습니다. 다음 개편은 메인 목록 화면부터 시작하는 것이 맞습니다.
