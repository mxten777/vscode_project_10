# BidSight 고객용 한 페이지 소개

이 문서는 클라이언트, 파트너, 내부 영업 담당자가 BidSight를 짧게 설명할 때 쓰는 1페이지 요약입니다.

## BidSight는 무엇인가

BidSight는 나라장터 공고를 자동으로 모으고, 필요한 공고만 빠르게 찾고, 놓치지 않게 알려주며, 참여 여부를 판단할 근거를 정리해 주는 입찰 지원 서비스입니다.

복잡한 기능을 많이 보여주는 제품이 아니라, 입찰 실무에서 반복되는 아래 문제를 줄이는 데 초점을 둡니다.

- 공고를 직접 뒤지는 시간
- 중요한 공고를 놓치는 문제
- 어떤 공고를 먼저 검토해야 할지 모호한 문제
- 참여 여부를 판단할 근거가 흩어져 있는 문제

## 누구를 위한 서비스인가

- 매일 나라장터 공고를 확인하는 입찰 담당자
- 영업 기회를 공공입찰에서 찾는 영업 담당자
- 여러 고객사의 공고를 대신 검토하는 컨설턴트
- 기관별, 업종별 흐름을 보고 팀의 검토 우선순위를 정해야 하는 관리자

## 클라이언트가 바로 이해해야 할 핵심 효익

| 효익 | 설명 |
|------|------|
| 검색 시간 절감 | 여러 공고를 직접 훑는 반복 작업을 줄입니다. |
| 공고 누락 감소 | 즐겨찾기와 알림으로 중요한 공고를 놓치지 않게 돕습니다. |
| 검토 우선순위 정리 | 기관, 업종, 예산 기준으로 먼저 볼 공고를 빠르게 추립니다. |
| 참여 판단 지원 | 낙찰 데이터와 분석 화면으로 검토 근거를 정리합니다. |

## 실제 사용 흐름은 이렇게 이해하면 됩니다

1. 시스템이 공고를 자동으로 수집합니다.
2. 사용자는 키워드, 기관, 업종, 예산으로 필요한 공고를 좁힙니다.
3. 중요한 공고는 즐겨찾기 하거나 알림 규칙으로 등록합니다.
4. 낙찰 데이터와 분석 화면을 보고 참여 여부를 판단합니다.
5. 팀은 같은 화면과 기준으로 검토 결과를 공유합니다.

## 지금 운영 기준으로 가능한 것

- 공고 검색과 필터링
- 공고 상세 조회
- 즐겨찾기
- 조건 기반 알림
- 분석 대시보드와 리포트
- 주기적 데이터 수집 운영

## 아직 과장해서 말하면 안 되는 것

- AI 추천 기능은 코드와 설계는 있으나 운영 연결이 완전히 닫히지 않았습니다.
- Stripe 기반 유료 결제 흐름은 운영 연결이 남아 있습니다.
- 따라서 현재 소개는 검색, 선별, 알림, 분석 중심으로 하는 편이 정확합니다.

## 이런 경우 특히 적합합니다

- 공고 검토량은 많은데 인력은 제한적이다
- 매번 비슷한 조건의 공고를 반복 검색한다
- 중요한 공고를 놓친 경험이 있다
- 입찰 참여 판단을 더 빠르게 해야 한다
- 팀 내에서 공고 검토 기준을 통일하고 싶다

## 이런 기대에는 바로 맞지 않을 수 있습니다

- 완전 자동 투찰가 산정만을 바로 기대하는 경우
- ERP, 그룹웨어, 외부 BI까지 한 번에 깊게 통합된 엔터프라이즈 제품을 기대하는 경우
- 오프라인 컨설팅 자체를 대체하는 서비스를 기대하는 경우

## 한 문장 소개가 필요할 때

BidSight는 공공입찰 공고를 자동으로 모으고, 필요한 공고만 빠르게 찾고, 놓치지 않게 알려주며, 참여 판단 근거까지 정리해 주는 입찰 지원 서비스입니다.

## 데모에서 보여주면 좋은 순서

1. 검색과 필터로 필요한 공고를 빠르게 좁히는 화면
2. 즐겨찾기와 알림으로 누락을 줄이는 흐름
3. 낙찰 데이터와 분석 화면으로 판단 근거를 보는 흐름
4. 대시보드와 리포트로 팀이 함께 보는 화면
