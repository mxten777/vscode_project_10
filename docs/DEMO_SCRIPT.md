# BidSight 시연 스크립트

이 문서는 데모 직전에 그대로 따라갈 수 있는 발표용 진행 스크립트입니다.

실제 공고 예시와 키워드 고정본은 `docs/DEMO_SCENARIOS.md` 를 기준으로 사용합니다.
리허설 진행과 점검은 `docs/REHEARSAL_CHECKLIST.md` 를 기준으로 사용합니다.

목표는 3가지입니다.

- 제품이 누구를 위한 것인지 빠르게 이해시킨다.
- 실제로 동작하는 흐름만 보여준다.
- 시연 중 리스크가 큰 기능은 과감히 뒤로 뺀다.

## 진행 원칙

시연은 기본적으로 `탑다운` 으로 진행합니다.

즉, 화면을 이곳저곳 점프하기보다 아래 순서로 내려갑니다.

1. 랜딩에서 서비스 가치 설명
2. 메인 검색에서 공고 선별
3. 공고 상세에서 판단 근거 확인
4. 즐겨찾기/알림에서 누락 방지 설명
5. 분석 대시보드에서 전체 흐름 정리

원칙은 간단합니다.

- 위 화면에서 아래 화면으로 내려간다.
- 아직 설명하지 않은 기능으로 먼저 점프하지 않는다.
- 청중이 "왜 이 다음 화면을 보는지" 이해되는 순서로만 이동한다.

## 1. 시연 한 줄 소개

BidSight는 공공입찰 공고를 자동으로 모으고, 필요한 공고만 빠르게 찾고, 놓치지 않게 알려주며, 참여 판단 근거를 정리해 주는 입찰 지원 서비스입니다.

## 2. 기본 시연 흐름

### Step 1. 문제 제기

이렇게 설명합니다.

"입찰 실무자는 매일 여러 공고를 직접 확인해야 하고, 어떤 공고를 먼저 봐야 할지 빠르게 판단하기 어렵습니다. BidSight는 그 반복 작업과 판단 부담을 줄이는 데 초점을 둡니다."

### Step 2. 랜딩에서 서비스 가치 설명

보여줄 화면:

- 공개 랜딩

강조할 포인트:

- 공고를 자동으로 모은다
- 필요한 공고만 빠르게 고른다
- 놓치지 않게 알림을 준다
- 판단 근거를 화면에 모아준다

말할 문장:

"이 서비스는 기능을 많이 나열하는 제품이 아니라, 공고 탐색 시간과 누락 리스크를 줄이는 데 집중합니다."

### Step 3. 검색과 선별 흐름 시연

보여줄 화면:

- 메인 대시보드
- 검색/필터
- 공고 리스트

강조할 포인트:

- 키워드, 기관, 업종, 예산 기준으로 빠르게 좁힐 수 있음
- 먼저 봐야 할 공고를 빠르게 추릴 수 있음

말할 문장:

"첫 번째 효익은 검색 자체보다 선별 속도입니다. 필요한 공고만 빨리 추리는 것이 핵심입니다."

### Step 4. 공고 상세와 판단 근거 시연

보여줄 화면:

- 공고 상세 페이지
- 낙찰 데이터
- 분석 관련 정보

강조할 포인트:

- 공고 세부 정보 확인
- 낙찰 이력과 관련 데이터 확인
- 참여 여부 판단에 필요한 근거 확인

말할 문장:

"단순 조회로 끝나는 것이 아니라, 이 공고를 검토할 가치가 있는지 판단할 근거까지 함께 보는 흐름입니다."

### Step 5. 즐겨찾기와 알림 시연

보여줄 화면:

- 즐겨찾기
- 알림 규칙 화면

강조할 포인트:

- 중요한 공고를 저장 가능
- 반복 검색 없이 조건 기반 추적 가능
- 공고 누락 리스크 감소

말할 문장:

"두 번째 핵심 효익은 누락 방지입니다. 좋은 공고를 찾는 것만큼, 다시 놓치지 않는 구조가 중요합니다."

### Step 6. 분석 대시보드 / 리포트 시연

보여줄 화면:

- 분석 대시보드
- 리포트

강조할 포인트:

- 기관/업종/지역 흐름을 한눈에 볼 수 있음
- 팀 차원의 검토 우선순위 정리에 도움

말할 문장:

"세 번째 효익은 의사결정 지원입니다. 개별 공고만 보는 것이 아니라, 전체 흐름 속에서 판단할 수 있습니다."

### Step 7. 마무리

이렇게 마무리합니다.

"정리하면 BidSight는 공고를 대신 모으고, 필요한 공고를 빠르게 고르고, 놓치지 않게 관리하고, 참여 판단 근거를 정리해 주는 서비스입니다."

## 3. 시연에서 강조할 메시지

- 검색 시간 절감
- 공고 누락 감소
- 검토 우선순위 정리
- 참여 판단 지원

## 4. 시연에서 과장하지 말아야 할 것

- AI 추천 기능이 이미 운영 핵심처럼 보이게 설명하지 않기
- Stripe 결제 흐름을 주요 완성 기능처럼 설명하지 않기
- 모든 기관/모든 경우에 최적 전략을 자동으로 준다고 말하지 않기

## 5. 시연 순서 요약

1. 문제 설명
2. 랜딩에서 서비스 가치 설명
3. 검색/필터로 공고 선별
4. 공고 상세에서 판단 근거 확인
5. 즐겨찾기와 알림으로 누락 방지 설명
6. 대시보드/리포트로 의사결정 지원 설명
7. 한 문장으로 마무리

이 순서는 곧 시연 화면의 `위에서 아래로 내려가는 흐름` 입니다.

## 6. 데모 전 체크리스트

- 공개 랜딩 문구 최종 확인
- 로그인 계정 준비
- 대표 검색 키워드 준비 (`docs/DEMO_SCENARIOS.md` 기준)
- 보여줄 공고 예시 2~3건 선정 (`docs/DEMO_SCENARIOS.md` 기준)
- 즐겨찾기/알림 예시 데이터 준비
- 분석 대시보드가 정상 표시되는지 확인
- 장애 시 대체 설명 문장 준비

## 7. 장애 시 우회 문구

### 수집 상태 질문을 받을 때

"현재는 수집 파이프라인을 운영 중이고, 시연에서는 실제 사용자 흐름 중심으로 보여드리겠습니다. 운영 안정화는 별도 단계로 계속 진행 중입니다."

### AI 추천 질문을 받을 때

"AI 관련 기능은 구조와 설계는 준비되어 있지만, 현재 시연에서는 검색, 알림, 분석처럼 이미 안정적으로 보여줄 수 있는 흐름에 집중하고 있습니다."

### 결제 질문을 받을 때

"결제는 상용화 단계에서 연결할 영역이고, 이번 시연의 핵심은 제품 가치와 실제 사용 흐름입니다."
