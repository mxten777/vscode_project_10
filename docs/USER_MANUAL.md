## 1. BidSight 소개

BidSight는 **대한민국 나라장터(조달청)** 공공 입찰 공고를 수집하고, 기업이 필요한 공고를 빠르게 찾고, 놓치지 않게 추적하며, 참여 판단 근거를 정리할 수 있도록 돕는 SaaS 플랫폼입니다.

| 항목 | 내용 |
|------|------|
| 서비스 URL | https://bid-platform.vercel.app |
| 지원 브라우저 | Chrome 115+, Edge 115+, Safari 16+, Firefox 115+ |
| 권장 해상도 | 1280 × 800 이상 |
| 모바일 지원 | 반응형 웹 (iOS Safari, Android Chrome) |

### 주요 기능

| 기능 | 설명 |
|------|------|
| 공고 검색 | 키워드·기관·업종·예산 중심으로 필요한 공고 선별 |
| 저장 검색 | 자주 쓰는 검색 조건 저장, 재적용, 수정, 삭제 |
| D-day 표시 | 마감 임박 공고 자동 강조 |
| 즐겨찾기 | 관심 공고 저장 후 다시 검토 |
| 알림 규칙 | 조건 기반 자동 알림 |
| 판단 보조 정보 | 유사 사례, 낙찰 데이터, 참여업체 정보 기반 검토 보조 |
| 유사공고 검색 | 관련 낙찰 사례와 비교 정보 조회 |
| 정밀 분석 | 참여업체 전체 목록 및 경쟁자 현황 |
| 분석 대시보드 | 기관·업종·지역별 낙찰 통계 및 트렌드 |

## 2. 시작하기 — 회원가입·로그인

### 2.1 회원가입

1. **https://bid-platform.vercel.app** 접속
2. 우상단 **[로그인]** 클릭
3. 로그인 페이지에서 **[회원가입]** 탭 선택
4. 이메일·비밀번호(6자 이상)·조직명 입력 후 **[회원가입]** 클릭
5. 가입이 완료되면 자동으로 로그인되고 메인 대시보드로 이동합니다.

회원가입이 완료되면 조직(org)이 자동 생성됩니다.

### 2.2 로그인

1. 로그인 페이지에서 이메일·비밀번호 입력
2. **[로그인]** 클릭 → 메인 대시보드로 이동

### 2.3 로그아웃 / 비밀번호 변경

- 헤더 우상단 **프로필 아이콘(이니셜 원형)** 클릭 → 드롭다운에서 **[로그아웃]** 또는 **[비밀번호 변경]** 선택

### 2.4 다크 모드

헤더 우상단 🌙/☀️ 아이콘 클릭으로 즉시 전환됩니다. 선택한 테마는 브라우저에 저장됩니다.

## 3. 메인 대시보드 — 공고 검색

### 3.1 화면 구성

```
┌──────── 헤더 (네비게이션 / 테마전환 / 프로필) ─────────┐
├────────────────────────────────────────────────────────┤
│  히어로 배너                                           │
│  [전체 공고 수]  [진행중 공고]  [마감임박]              │
├────────────────────────────────────────────────────────┤
│  카테고리 칩                                           │
│  [전체] [소프트웨어] [건설·공사] [용역·서비스] [물품] ...│
├────────────────────────────────────────────────────────┤
│  ⚡ 트렌딩 키워드 칩                                    │
├────────────────────────────────────────────────────────┤
│  검색창   [상태 ▽]  [정렬 ▽]                           │
├────────────────────────────────────────────────────────┤
│  공고 카드 목록                                         │
├────────────────────────────────────────────────────────┤
│  페이지네이션                                          │
└────────────────────────────────────────────────────────┘
```

### 3.2 검색 및 필터

#### 키워드 검색
검색창에 키워드를 입력하면 300ms 디바운스 후 자동으로 필터링됩니다.
- 예: `AI 플랫폼`, `정보화시스템`, `유지보수`

#### 카테고리 칩
히어로 배너 하단 칩을 클릭해 업종별로 빠르게 필터링합니다.

| 칩 | 포함 공고 |
|----|----------|
| 전체 | 모든 공고 |
| 소프트웨어 | SW개발, AI, 클라우드, 정보화시스템 |
| 건설·공사 | 건설, 공사, 시공 |
| 용역·서비스 | 용역, 컨설팅, 위탁 |
| 물품·장비 | 구매, 납품, 장비 |
| 시설관리 | 시설, 유지보수, 청소 |

#### 트렌딩 키워드
히어로 배너 내 플로팅 키워드 칩 클릭 시 해당 키워드로 즉시 검색됩니다.

#### 상태 필터
- **진행중 (OPEN)**: 현재 입찰 가능한 공고
- **마감 (CLOSED)**: 마감된 공고
- **낙찰완료 (RESULT)**: 낙찰 결과가 공시된 공고

> 💡 여러 필터를 동시에 적용할 수 있습니다.

### 3.3 공고 카드 읽기

| 항목 | 설명 |
|------|------|
| 공고명 | 입찰 공고 제목 |
| 발주기관 | 공고를 게시한 기관명 |
| 예산 | 추정 예산 금액 |
| 마감일 | 입찰 마감 날짜 |
| D-day 뱃지 | 🔴 D-3 이내: 긴급 / 🟠 D-7 이내: 주의 |
| 상태 뱃지 | `진행중` / `마감` / `낙찰완료` |
| ★ 즐겨찾기 | 클릭 시 즐겨찾기 추가/제거 |

### 3.4 페이지네이션

카드 보기에서는 한 페이지 최대 20건, 테이블 보기에서는 최대 50건이 표시됩니다. 하단 페이지 번호 또는 이전/다음 버튼으로 이동합니다.

### 3.5 저장 검색

메인 화면에서는 현재 검색 조건을 그대로 저장할 수 있습니다.

1. 키워드 또는 상태를 먼저 선택합니다.
2. 검색 영역의 **저장** 동작을 눌러 이름을 입력합니다.
3. 저장한 검색 목록에서 다시 불러오거나, 이름을 수정하거나, 삭제할 수 있습니다.

저장 검색으로 할 수 있는 작업:

| 기능 | 설명 |
|------|------|
| 다시 적용 | 저장해 둔 키워드/상태/정렬을 즉시 복원 |
| 수정 | 이름과 검색 조건을 다시 저장 |
| 삭제 | 더 이상 쓰지 않는 조건 제거 |
| 알림 전환 | 키워드 또는 상태가 있는 저장 검색을 알림 규칙으로 바로 전환 |

> 저장 검색은 계정별로 서버에 저장되며, Free 플랜 기준 최대 8개까지 보관됩니다.

## 4. 공고 상세 조회

공고 카드 클릭 시 상세 페이지로 이동합니다.

### 4.1 헤더 영역

- **공고명**, **상태 뱃지**, **계약방식 뱃지**
- **분석 레벨 뱃지**: `후보군` (레벨 2) 또는 `정밀 분석` (레벨 3)
- **즐겨찾기 버튼**: 노란별(등록됨) / 빈별(미등록), 로그인 필요

### 4.2 D-day 카운트다운

마감일까지 남은 일수를 실시간으로 표시합니다.

| 잔여일 | 표시 방식 |
|--------|----------|
| 오늘 마감 | 🔴 **D-DAY** (빨간 강조) |
| 1~3일 | 🔴 D-1 ~ D-3 (긴급) |
| 4~7일 | 🟠 D-4 ~ D-7 (주의) |
| 8일 이상 | ⬜ 일반 표시 |
| 마감됨 | 마감 처리 |

### 4.3 참여 판단 요약

상세 화면 상단에는 공고 기본 정보보다 먼저 아래 판단 요소가 요약되어 표시됩니다.

- 지금 상태와 마감 시점
- 예산 규모
- 결과 데이터 또는 추가 검토 필요 여부
- 다음 행동 제안

### 4.4 기본 정보

| 항목 | 내용 |
|------|------|
| 발주기관 | 기관명 |
| 추정예산 | 원 단위 금액 (\₩ 포맷) |
| 마감일시 | 날짜 및 시각 |
| 지역 | 지역명 또는 코드 |
| 업종 | 업종 분류명 |
| 계약방식 | 입찰 방식 |

### 4.5 낙찰 결과

공고 상태가 `낙찰완료(RESULT)` 이고 낙찰 데이터가 수집된 경우 표시됩니다.

| 항목 | 내용 |
|------|------|
| 낙찰업체 | 낙찰 기업명 |
| 낙찰금액 | \₩ 포맷 |
| 낙찰률 | % (예: 97.3%) |
| 개찰일 | 날짜 및 시각 |

### 4.6 원본 데이터

카드 하단 **[원본 데이터 보기]** 를 펼치면 나라장터에서 수집한 원본 JSON 데이터를 확인할 수 있습니다.

### 4.7 원본 데이터 확인

공고 상세 페이지 하단에서 **원본 데이터** 영역을 펼쳐 수집된 원문 JSON을 확인할 수 있습니다.

## 5. AI 투찰가 추천 (연결된 환경에서만 제공)

> 현재 공개 운영 환경에서는 AI 서비스 연결 상태에 따라 이 섹션이 보이지 않거나 준비중으로 표시될 수 있습니다.

**분석 레벨 2 이상** 공고 (마감 7일 이내, 예산 1억 이상, 또는 낙찰결과 있는 공고)에서 제공됩니다.

### 5.1 3가지 전략 추천

| 전략 | 설명 | 특징 |
|------|------|------|
| 🛡 **보수적** | 안전 마진 확보 위주 | 낙찰률 낮지만 손실 위험 최소 |
| 🎯 **기준** | 시장 평균 낙찰가 기반 | 균형잡힌 전략 (권장) |
| ⚡ **공격적** | 최저가 경쟁 | 낙찰 가능성 높이나 마진 낮음 |

각 전략 카드에는 **추천 투찰가(₩), 예상 낙찰률(%), 전략 설명**이 표시됩니다.

### 5.2 유사 낙찰 사례

AI 투찰가 추천 섹션 하단에서 **벡터 유사도** 기반으로 유사한 과거 공고의 낙찰 사례를 최대 5건 조회합니다.

| 항목 | 내용 |
|------|------|
| 공고명 | 유사 공고 제목 (클릭 시 해당 공고 상세로 이동) |
| 낙찰금액 | 실제 낙찰된 금액 |
| 낙찰률 | 예정가 대비 낙찰 비율 |
| 낙찰업체 | 낙찰 기업명 |
| 개찰일 | 낙찰 날짜 |

> 💡 유사 사례가 많을수록 AI 추천의 정확도가 높아집니다. `데이터 품질` 배지로 추천 신뢰도를 확인하세요.

### 5.3 데이터 품질 배지

| 배지 | 의미 |
|------|------|
| 🟢 **실데이터** | 실제 낙찰 사례 기반, 높은 신뢰도 |
| 🟡 **부분 데이터** | 일부 추정 포함 |
| 🔴 **데이터 부족** | 유사 사례 없음, 참고용으로만 사용 |

## 6. 정밀 분석 — 참여업체 현황 (분석 레벨 3)

**정밀 분석**은 입찰에 참여한 모든 업체의 투찰 내역을 조회하는 기능으로, 가장 심층적인 경쟁자 분석을 제공합니다.

### 6.1 정밀 분석 활성화 조건

다음 중 하나에 해당하는 공고에서 참여업체 데이터를 자동 수집합니다:

| 조건 | 설명 |
|------|------|
| 즐겨찾기 등록 | 공고를 즐겨찾기에 추가하면 즉시 수집 트리거 |
| 마감 7일 이내 | 마감이 임박한 공고 (레벨 2 이상) |
| 예산 1억 이상 | 고가 공고 |
| 분석 레벨 2+ | 후보군으로 분류된 공고 |

### 6.2 참여업체 목록

마감/낙찰완료 공고에서 데이터 수집 후 표시됩니다.

| 항목 | 내용 |
|------|------|
| 순위 | 투찰가 순위 (1위 = 낙찰자) |
| 업체명 | 입찰 참여 업체명 |
| 투찰금액 | ₩ 포맷 |
| 투찰률 | 예정가 대비 비율 (%) |
| 낙찰 여부 | 🏆 아이콘으로 낙찰자 표시 |

### 6.3 정밀 분석 카드

- **헤더**: `정밀 분석` 배지 (Microscope 아이콘 + 초록색)
- 참여업체 수집 전: "즐겨찾기 추가 시 자동 수집됩니다" 안내 문구
- 수집 완료 시: 전체 참여업체 순위표 표시

> ℹ️ 데이터 수집에는 몇 분이 소요될 수 있습니다. 페이지를 새로고침하면 최신 데이터로 갱신됩니다.

## 7. 즐겨찾기

### 7.1 추가 방법

- **공고 카드** 우상단 ★ 아이콘 클릭
- **공고 상세 페이지** 헤더의 **[즐겨찾기 추가]** 버튼 클릭

> 로그인이 필요합니다. 즐겨찾기 추가 시 해당 공고는 **분석 레벨 3(정밀 분석)** 으로 자동 승격되어 참여업체 데이터 수집이 시작됩니다.

### 7.2 즐겨찾기 목록

상단 네비게이션 **[즐겨찾기]** 클릭 → 등록된 공고 전체 목록 표시

각 카드에서:
- **[상세보기]**: 해당 공고 상세 페이지로 이동
- **★ 아이콘 재클릭**: 즐겨찾기 제거

### 7.3 주의 사항

- 즐겨찾기 데이터는 계정에 저장되므로 다른 기기에서도 동일하게 표시됩니다.
- 플랜 제한은 조직 기준으로 적용됩니다. Free는 최대 50개, Pro/Enterprise는 사실상 제한 없이 사용할 수 있습니다.

## 8. 알림 규칙 관리

### 8.1 알림이란?

조건을 등록해 두면 조건에 맞는 새 공고가 수집될 때 자동으로 **이메일**을 발송합니다.

### 8.2 알림 규칙 생성

1. 상단 네비게이션 **[알림]** 클릭
2. 우상단 **[+ 규칙 추가]** 버튼 클릭
3. 아래 항목 입력 후 **[저장]**

| 항목 | 설명 | 예시 |
|------|------|------|
| 규칙 이름 | 규칙을 구분하기 위한 이름 (선택) | `AI 플랫폼 공고 알림` |
| 유형 | 키워드 또는 필터 규칙 | `KEYWORD` / `FILTER` |
| 키워드 | 공고명에 포함되어야 할 단어. 여러 개면 공백으로 구분 | `AI`, `클라우드`, `데이터` |
| 공고 상태 | 진행중/마감/결과발표 중 특정 상태만 추적 | `OPEN` |
| 최소 예산 | 예산 하한 (원 단위) | `50000000` |
| 최대 예산 | 예산 상한 (원 단위) | `1000000000` |
| 알림 채널 | 현재 실동작 기준은 이메일 | 이메일 |

> 메인 화면의 저장 검색에서도 알림 규칙을 바로 만들 수 있습니다. 키워드나 상태가 있는 저장 검색만 전환 가능합니다.

### 8.3 알림 규칙 관리

| 기능 | 방법 |
|------|------|
| 활성화/비활성화 | 규칙 카드의 토글 스위치 클릭 |
| 수정 | 규칙 카드 **[편집]** 버튼 |
| 삭제 | 규칙 카드 **[삭제]** 버튼 |

### 8.4 알림 발송 이력

**[발송 이력]** 탭에서 최근 알림 발송 기록(발송일시, 규칙명, 매칭 공고명, 결과)을 확인할 수 있습니다.

### 8.5 알림 발송 시각

공고 수집은 현재 **평일 오전 09:00 KST** 기준 오케스트레이터에서 실행되고, 알림 평가는 **평일 오전 11:00 KST 전후**에 처리됩니다. 신규 공고가 없으면 알림은 발송되지 않습니다.

현재 운영 기준:

- 실동작 채널은 **이메일**입니다.
- 추가 채널 표시는 있을 수 있지만 운영 연결은 아직 준비 중입니다.
- Free 플랜은 최대 3개, Pro는 최대 50개, Enterprise는 사실상 제한 없이 알림 규칙을 만들 수 있습니다.

## 9. 분석 대시보드 (Analytics)

상단 네비게이션에서 접근할 수 있는 **분석 대시보드**는 나라장터 낙찰 데이터 기반의 시장 통계를 제공합니다.

### 9.1 KPI 요약 카드

페이지 상단에 4가지 핵심 지표가 표시됩니다.

| KPI | 내용 |
|-----|------|
| 총 공고 | 수집된 전체 공고 수 |
| 낙찰 건수 | 낙찰 처리된 공고 수 |
| 참여 기관 | 발주기관 총 수 (실데이터 기준) |
| 평균 낙찰률 | 전체 낙찰률 평균 (%) |

### 9.2 분석 탭

4개 탭으로 구성됩니다.

#### 📊 전체 통계 탭

- **월별 낙찰 건수**: 월별 공고 · 낙찰 추이 바 차트
- **낙찰률 분포**: 낙찰률 구간별 빈도 차트

#### 🏛 기관별 분석 탭

발주기관별 낙찰 통계 (TOP 기관 순위, 평균 낙찰률)

#### 🏭 업종별 분석 탭

업종 카테고리별 공고 건수 및 평균 예산 분포

#### 🗺 지역별 분석 탭

지역별 공고 분포 및 평균 낙찰 금액

### 9.3 데이터 품질 표시

각 분석 탭 우상단에 **데이터 품질 배지**가 표시됩니다.

| 배지 | 의미 |
|------|------|
| 🟢 실데이터 | 나라장터 실제 낙찰 데이터 100% 반영 |
| 🟡 부분 데이터 | 일부 기간 데이터 누락 |
| 🔴 준비중 | 아직 충분한 데이터가 없음 |

### 9.4 수집 현황 카드

분석 대시보드 내 **수집 현황** 카드에서 공고·낙찰·분석 데이터의 마지막 수집 일시와 현황을 확인할 수 있습니다.

## 13. 자주 묻는 질문 (FAQ)

**Q1. 공고 데이터는 얼마나 자주 업데이트되나요?**  
A. 평일 오전 09:00 KST에 나라장터에서 자동 수집됩니다. 낙찰 데이터는 오전 09:10 KST에 추가 수집됩니다.

**Q2. 알림은 언제 발송되나요?**  
A. 평일 오전 11:00 KST경 발송됩니다.

**Q3. AI 투찰가 추천이 보이지 않아요.**  
A. 분석 레벨 2 이상 공고에서만 대상이 되며, 현재 운영 환경에 AI 서비스가 연결되지 않은 경우에는 보이지 않거나 준비중으로 표시될 수 있습니다.

**Q4. 정밀 분석(참여업체)이 표시되지 않아요.**  
A. 즐겨찾기에 추가하거나 레벨 2 조건을 만족해야 합니다. 데이터 수집 후 반영까지 몇 분이 소요됩니다.

**Q5. 즐겨찾기 개수 제한이 있나요?**  
A. 조직 플랜 기준으로 Free는 최대 50개, Pro/Enterprise는 사실상 제한 없이 사용할 수 있습니다.

**Q6. 알림 규칙은 몇 개까지 만들 수 있나요?**  
A. 조직 플랜 기준으로 Free는 3개, Pro는 50개, Enterprise는 사실상 제한 없이 만들 수 있습니다.

**Q7. 나라장터 원문을 보고 싶은데 어디서 볼 수 있나요?**  
A. 공고 상세 페이지 내 **[나라장터에서 보기]** 버튼을 클릭하세요.

**Q8. 모바일에서도 사용할 수 있나요?**  
A. 반응형 웹으로 설계되어 스마트폰·태블릿에서도 사용 가능합니다.

**Q9. 비밀번호를 잊었어요.**  
A. 로그인 페이지 하단 **[비밀번호 찾기]** 링크를 클릭하면 이메일로 재설정 링크가 발송됩니다.

**Q10. 팀 멤버를 초대하려면 어떻게 하나요?**  
A. **[설정]** → **[팀 관리]** → **[멤버 초대]** 에서 이메일을 입력해 초대 메일을 발송합니다.

**Q11. 분석 대시보드의 데이터가 최신이 아닌 것 같아요.**  
A. 분석 캐시는 매일 오전 05:00 KST에 재계산됩니다. 최신 낙찰 데이터가 반영되기까지 하루가 걸릴 수 있습니다.
