## 2. 공통 레이아웃

### 2.1 Header (네비게이션 바)

```
┌────────────────────────────────────────────────────────────┐
│  🏛 입찰 분석 플랫폼    [공고]  [즐겨찾기]  [알림]  [리포트] │
│                                                    [로그인] │
└────────────────────────────────────────────────────────────┘
```

**데스크톱**:
- 좌측: 로고 + 앱 이름 (홈 링크)
- 중앙: 네비게이션 링크 (공고, 즐겨찾기, 알림, 리포트)
- 우측: 인증 상태 (로그인 버튼 / 사용자 이메일 + 로그아웃)

**모바일** (반응형):
- 좌측: 로고
- 우측: 햄버거 메뉴 (Sheet 컴포넌트)
  - 메뉴 펼침 시 세로 네비게이션 + 인증 버튼

**컴포넌트**: `src/components/header.tsx`

### 2.2 컨텐츠 영역

```
┌────────────────────────────────────────────────┐
│  Header (fixed top)                            │
├────────────────────────────────────────────────┤
│                                                │
│  <main className="container mx-auto py-6">    │
│    {children}                                  │
│  </main>                                       │
│                                                │
└────────────────────────────────────────────────┘
```

- 최대 너비: `container` (기본 `max-w-7xl`)
- 좌우 패딩: `px-4 md:px-6`
- 상단 여백: `py-6`

## 3. 페이지별 상세 설계

### 3.1 로그인/회원가입 페이지

**경로**: `/login`
**파일**: `src/app/login/page.tsx`
**인증**: 불필요 (로그인 상태 접근 시 `/` 리다이렉트)

```
┌────────────────────────────────────────────┐
│            입찰·조달 분석 플랫폼             │
│     AI 기반 맞춤 입찰 정보를 받아보세요       │
│                                            │
│  ┌──────────────────────────────────────┐  │
│  │  [로그인]  [회원가입]    ← 탭 전환    │  │
│  ├──────────────────────────────────────┤  │
│  │                                      │  │
│  │  이메일: [________________]          │  │
│  │  비밀번호: [________________]        │  │
│  │  (조직명): [________________]  *가입  │  │
│  │                                      │  │
│  │  [        로그인 / 가입하기        ]  │  │
│  │                                      │  │
│  └──────────────────────────────────────┘  │
└────────────────────────────────────────────┘
```

**UI 구성요소**:
- **Card**: 중앙 정렬 카드 (`max-w-md`)
- **Tabs**: "로그인" / "회원가입" 탭 전환
- **Input**: 이메일, 비밀번호 필드 (Zod 실시간 검증)
- **Button**: 제출 버튼 (로딩 상태 표시)
- **Toast**: 성공/실패 메시지 (sonner)

**동작**:
1. 로그인 성공 → `redirect` 쿼리 파라미터가 있으면 해당 경로로, 없으면 `/`
2. 회원가입 → 자동으로 org 생성 + 로그인 → 메인 페이지

```
│                                                         │
│  ┌──────────────────────────────────────────────────┐   │
│  │ 제목           │ 기관    │ 예산      │ 마감일  │상태│   │
│  │────────────────┼────────┼──────────┼────────┼───│   │
│  │ 서울시 정보...  │ 서울시  │ ₩5억     │ 02/15  │🟢│   │
│  │ 도로공사 유지... │ 한국도로│ ₩12억    │ 02/20  │🟢│   │

│  └──────────────────────────────────────────────────┘   │
│                                                         │
│  [◀ 이전]  1  2  3  ...  10  [다음 ▶]                   │
│                                                         │
│  총 1,523건                                              │
└─────────────────────────────────────────────────────────┘
```

**UI 구성요소**:
- **검색바** (`Input`): 300ms 디바운스, URL searchParams 동기화
- **필터** (`Select`): 상태(전체/진행중/마감/결과발표), 정렬 기준, 정렬 순서
- **테이블** (`Table`): 반응형, 행 클릭 → 상세 페이지 이동
- **Badge**: 상태 뱃지 (OPEN=파란/CLOSED=빨간/RESULT=초록)
- **Pagination**: 페이지 번호 + 이전/다음 버튼
- **Skeleton**: 로딩 상태 시 테이블 스켈레톤

**상태 관리**:
```typescript
// URL 파라미터 ↔ 검색 상태 동기화
const searchParams = useSearchParams();

// 검색어 디바운스
const [localQ, setLocalQ] = useState(q);
useEffect(() => {
  const timer = setTimeout(() => updateParams({ q: localQ, page: "1" }), 300);
  return () => clearTimeout(timer);
}, [localQ]);
```

## 4. 컴포넌트 계층 구조

```
RootLayout (layout.tsx)
├── Providers (QueryClient, Tooltip, Toaster)
├── Header
│   ├── Logo (Link → /)
```

## 7. 상태 표시 규칙

### 7.1 공고 상태 뱃지

| 상태 | 색상 | 라벨 |
|---|---|---|
| `OPEN` | 파란색 (`default`) | 진행중 |
| `CLOSED` | 빨간색 (`destructive`) | 마감 |
| `RESULT` | 초록색 (`secondary`) | 결과발표 |

### 7.2 알림 발송 상태

| 상태 | 아이콘 | 색상 |
|---|---|---|
| `SENT` | ✅ | 초록 |
| `FAIL` | ❌ | 빨강 |

### 7.3 금액 표시

- 포맷: `new Intl.NumberFormat("ko-KR", { style: "currency", currency: "KRW" })`
- 예시: `₩500,000,000`
- NULL: `-` 표시

### 7.4 날짜 표시

- 전체: `YYYY-MM-DD HH:mm`
- 요약: `MM/DD` (테이블)
- NULL: `-` 또는 "미정" 표시
